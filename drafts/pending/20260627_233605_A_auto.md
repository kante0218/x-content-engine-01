Claude にMCPで Slack も Gmail も Playwright も繋いだら、一番ハマったのは「権限を絞らないと事故る」でした。

最初は全部のスコープ開放で繋いで、テストで「未読メール要約して」って投げたら、Claude が勝手に既読化のツールまで叩いて受信箱を全部既読にしかけました。あれは肝が冷えました。

回避策はシンプルで、MCPサーバ側で read 系と write 系を別サーバに分けて、Gmail は基本 read-only スコープでしか繋がない。送信や既読化が必要なときだけ別MCPを明示的に有効化する。Claude Code なら `.mcp.json` でプロジェクト単位に持たせて、危ない方はデフォルトで読み込まない構成にしてます。

Playwright MCP は逆に強烈で、これだけでブラウザ操作が丸ごとエージェントの手足になります。Slackの管理画面みたいにAPIが面倒な操作を、Claude に「この画面開いてこのボタン押して」で任せられる。受託の検証作業がごっそり消えました。

ただ手放しでは使えなくて、hooks で write 系ツールの実行前に確認を挟むのは絶対やった方がいいです。PreToolUse で対象ツール名をフックして、send とか delete が含まれる呼び出しは一回止める。これ入れてから事故がゼロになりました。

MCPは「全部繋げば賢くなる」じゃなくて、繋いだ瞬間にエージェントが現実世界に手を出せるようになる、という話だと思ってます。便利さと事故が完全に表裏一体で、結局ここでも要件定義する側の責任が残るんですよね。
